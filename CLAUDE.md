# PAX Marketplace

## What This Is
A Hugo static site serving as the public catalog for PAX (Portable Analytical eXpertise) — the composable knowledge packages from the [Praxis](https://github.com/JELambert/praxis) research infrastructure framework.

**Live at [pax-market.com](https://pax-market.com)**

## Tech Stack
- **Hugo** static site generator (no theme — custom layouts)
- **Proxmox CT 110** (192.168.68.110) — nginx serving the built site
- **Cloudflare Named Tunnel** — pax-market.com via tunnel `praxis-marketplace`
- **nginx-proxy-manager** (CT 107) — routes marketplace.home locally
- **GitHub Pages** backup deployment via GitHub Actions
- **Vanilla JS** for client-side search/filter
- **Python** sync script to generate content from PAX directories

## Project Structure
```
content/packs/<name>/index.md  — Hugo content pages (generated by sync-pax.py)
layouts/                       — Hugo templates (baseof, pack list, pack detail, partials)
scripts/sync-pax.py            — Reads ../praxis/pax/ dirs, generates content + archives
scripts/deploy.sh              — One-command build + deploy to Proxmox
scripts/rebuild.sh             — CT 105-local rebuild (sync published + hugo + rsync)
static/packs/                  — .pax.tar.gz download archives
data/registry.json             — Machine-readable PAX index (schema v2.0)
data/constructs.json           — Cross-PAX construct index
.github/workflows/deploy.yml   — GitHub Actions → GitHub Pages deployment
```

## Key Commands
- `hugo server` — Local dev server at localhost:1313
- `./scripts/deploy.sh` — Full sync + build + deploy to Proxmox
- `./scripts/deploy.sh --build-only` — Build + deploy (skip pack sync)
- `./scripts/deploy.sh --url` — Show marketplace URL
- `python3 scripts/sync-pax.py` — Regenerate content from praxis PAX dirs

## Agent Publishing
Agents publish via MCP tools on CT 105:
```
praxis_create_pax("name", ...)     → scaffold PAX
praxis_publish_pax("name")         → validate + publish + rebuild marketplace
praxis_unpublish_pax("name")       → remove from marketplace
```
Publishing triggers `rebuild.sh` on CT 105 which runs sync-pax.py (--published-only), Hugo, and rsync to CT 110.

## Deployment
- **Public:** https://pax-market.com (Cloudflare Named Tunnel)
- **LAN:** http://192.168.68.110
- **Infrastructure:** Proxmox main (192.168.68.70), CT 105 = praxis, CT 110 = marketplace, CT 107 = nginx-proxy-manager

## Content Pipeline
PAX content pages are **generated, not hand-edited**. The source of truth is `pax.yaml` in each PAX directory. The rebuild pipeline on CT 105 only syncs packs in the `pax_publications` table with status='published'.

## Design Decisions
- All CSS is inline in baseof.html (no build tools, no Tailwind dependency)
- Dark theme with semantic color coding per PAX type
- Client-side search reads data attributes from rendered HTML cards
- registry.json (schema v2.0) serves as stable API for agent consumption
- baseURL is `https://pax-market.com/` — hugo server overrides for local dev
