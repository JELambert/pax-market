# Praxis Marketplace

## What This Is
A Hugo static site serving as the public catalog for PAX (Portable Analytical eXpertise) packs — the composable knowledge packages from the [Praxis](https://github.com/praxis-research/praxis) research infrastructure framework.

## Tech Stack
- **Hugo** static site generator (no theme — custom layouts)
- **Proxmox CT 110** (192.168.68.110) — nginx serving the built site
- **Cloudflare Quick Tunnel** — public URL for demo access (changes on restart)
- **nginx-proxy-manager** (CT 107) — routes `marketplace.home` to CT 110
- **GitHub Pages** backup deployment via GitHub Actions
- **Vanilla JS** for client-side search/filter
- **Python** sync script to generate content from PAX directories

## Project Structure
```
content/packs/<name>/index.md  — Hugo content pages (generated by sync-pax.py)
layouts/                       — Hugo templates (baseof, pack list, pack detail, partials)
scripts/sync-pax.py            — Reads ../praxis/pax/ dirs, generates content + archives
scripts/deploy.sh              — One-command build + deploy to Proxmox
static/packs/                  — .pax.tar.gz download archives
data/registry.json             — Machine-readable pack index (generated by sync-pax.py)
.github/workflows/deploy.yml   — GitHub Actions → GitHub Pages deployment
```

## Key Commands
- `hugo server` — Local dev server at localhost:1313
- `./scripts/deploy.sh` — Full sync + build + deploy to Proxmox
- `./scripts/deploy.sh --build-only` — Build + deploy (skip pack sync)
- `./scripts/deploy.sh --url` — Show current public tunnel URL
- `python3 scripts/sync-pax.py` — Regenerate content from praxis PAX dirs

## Deployment
- **LAN:** http://192.168.68.110
- **Public:** Cloudflare Quick Tunnel (run `./scripts/deploy.sh --url` to get current URL)
- **Infrastructure:** Proxmox main (192.168.68.70), CT 110 = marketplace, CT 107 = nginx-proxy-manager

The Cloudflare tunnel URL changes on container/service restart. For a stable URL, set up a named Cloudflare tunnel with a custom domain.

## Content Pipeline
Pack content pages are **generated, not hand-edited**. The single source of truth is `pax.yaml` in each PAX directory. To update pack content:
1. Modify the PAX in the praxis repo (or sync from CT 105)
2. Run `./scripts/deploy.sh` (syncs packs, builds, deploys)

PAX packs on the Praxis container (CT 105) can be synced to local with:
```bash
sshpass -p "vaultlock1" ssh root@192.168.68.70 \
  "pct exec 105 -- tar czf - -C /opt/praxis/pax ." | \
  tar xzf - -C ../praxis/pax/
```

## Design Decisions
- All CSS is inline in baseof.html (no build tools, no Tailwind dependency)
- Dark theme with semantic color coding per PAX type
- Client-side search reads data attributes from rendered HTML cards
- registry.json serves as stable schema for future agent API consumption
- baseURL is `/` in hugo.toml — GitHub Actions overrides it at deploy time
