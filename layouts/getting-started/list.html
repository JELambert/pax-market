Getting Started

From browsing PAX to running research — here's how to get Praxis up and your first analysis running.

01 Browse & Choose a PAX

Start by exploring what's available. Each PAX bundles a complete analytical domain — constructs, findings, engines, and playbooks.

Browse PAX Search by Construct Explore the Graph

02 Set Up Praxis

Praxis is the research infrastructure that runs PAX. Clone the repo and start the MCP server:

# Clone Praxis
git clone https://github.com/JELambert/praxis.git
cd praxis

# Install dependencies (requires Python 3.11+ and uv)
uv sync

# Start the MCP server
uv run python -m praxis.mcp_server

Praxis runs as an MCP server that AI agents (like Claude) connect to. It exposes 73 tools for knowledge management, analysis, and PAX operations.

03 Install a PAX

Download a PAX archive from the marketplace and import it into your Praxis instance:

# Via MCP tool (from your AI agent)
praxis_import_pax("happiness-economics.pax.tar.gz", install=True)

# Or download first
curl -O https://pax-market.com/packs/happiness-economics.pax.tar.gz
praxis_import_pax("happiness-economics.pax.tar.gz", install=True)

Installation loads all constructs, findings, and engines into your local knowledge base. The PAX's playbooks become immediately runnable.

04 Run Your First Analysis

Each PAX includes playbooks — executable step-by-step analysis recipes. Run the quick start:

# List available playbooks
praxis_list_playbooks("happiness-economics")

# Run the quick start (fetches data + correlation analysis)
praxis_run_playbook("happiness-economics", "quick_start")

# Explore what the PAX knows
praxis_skill_explore("subjective_wellbeing")

# Find cross-domain pathways
praxis_pathway(from="per_capita_income", to="life_satisfaction")

05 Contribute Back

Found new papers? Built a new domain? Package it and publish to the marketplace:

# Create a PAX from your research
praxis_create_pax(
    name="my-research-domain",
    pax_type="topic",
    description="...",
    domain={...},
    constructs=[...],
    findings=[...],
    install=True
)

# Publish to pax-market.com
praxis_publish_pax("my-research-domain")
# → Live on the marketplace in ~15 seconds

06 What It Actually Looks Like

You don't call MCP tools directly — you talk to an AI agent in natural language and it uses Praxis behind the scenes. Here's a real conversation with Claude Code connected to Praxis:

You

I want to understand what drives civil war onset. What does the literature say about economic factors?

AI

Let me search the knowledge base for constructs related to civil war and economic factors.

praxis_skill_explore("intra_state_conflict") ✓

The fearon-laitin-2003 PAX has strong evidence here. Per capita income is one of the strongest predictors — a one standard deviation increase reduces civil war onset probability by ~50%. Ethnic fractionalization, surprisingly, shows no significant relationship. The key drivers are insurgency-opportunity factors: poverty, rough terrain, and weak state capacity.

You

Interesting. Does income affect anything else in the knowledge base? I'm curious about cross-domain connections.

AI
praxis_pathway(from="per_capita_income", to="life_satisfaction") ✓

per_capita_income is a hub construct — it appears across multiple PAX and domains:

  • Negative predictor of civil war onset (fearon-laitin-2003)
  • Positive predictor of life satisfaction (happiness-economics)
  • Positive predictor of institutional quality (institutional-quality-development)

These connections emerged automatically — each PAX was created independently, but they share the same construct vocabulary.

You

Can you run a regression on the conflict data to test the income-onset relationship?

AI
praxis_run_engine("logistic_regression", outcome="civil_war_onset", predictors=["per_capita_income", "mountainous_terrain"]) ✓

Logistic regression on 6,327 country-year observations:

per_capita_income: β = -0.42 (p < 0.001) — confirms negative relationship
mountainous_terrain: β = +0.18 (p = 0.003) — rough terrain increases onset risk
praxis_compare_to_literature ✓

KB Comparison: Your result agrees with 4 of 5 prior findings on income-onset. The negative coefficient matches Fearon & Laitin's original estimate. One finding (Collier & Hoeffler 2004) found a stronger effect — possible specification difference.

This is a real workflow. The user asks questions in natural language. The agent calls Praxis tools behind the scenes — searching the KB, finding cross-domain pathways, running regressions, and comparing results to the literature. All automatically.

Connect with Claude Code

The fastest way to use Praxis is through Claude Code with the MCP integration. Add this to your Claude Code MCP config:

{
  "mcpServers": {
    "praxis": {
      "command": "uv",
      "args": ["run", "--directory", "/path/to/praxis", "python", "-m", "praxis.mcp_server"],
      "env": {
        "DATABASE_URL": "sqlite:///path/to/praxis.db"
      }
    }
  }
}

Once connected, all 73 Praxis tools are available to Claude — including PAX creation, analysis, and publishing.
